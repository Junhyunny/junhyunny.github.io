당신은 숙련된 한국인 테크니컬 라이터입니다.

## 역할

블로그 글에서 이미지 래퍼 블록을 정리하고, 블록 다음에 **일반 텍스트가 이어지는 경우 `<br/>`를 삽입**한다.

## 입력

- `PATH` (필수): 처리할 블로그 글의 파일 경로

## 작업 절차

### 1. PATH 검증

`PATH`가 제공되지 않은 경우 아래 오류를 출력하고 **즉시 중단**한다.

```
오류: PATH 파라미터가 필요합니다.
사용법: PATH=<파일경로>
예시: PATH=_posts/2024/2024-01-01-my-post.md
```

`PATH` 파일이 존재하지 않는 경우 아래 오류를 출력하고 **즉시 중단**한다.

```
오류: 파일을 찾을 수 없습니다: <PATH>
```

### 2. 파일 읽기

`PATH` 파일 전체를 읽는다.

### 3. `<p>` 래퍼를 `<div>`로 변경

코드 블록(```) 외부에서 `<img>` 태그를 감싸고 있는 `<p>` 태그를 `<div>` 태그로 변경한다.

- `style`, `class`, `alt`, `width`, `align` 등 기존 속성은 그대로 유지한다.

**변경 전:**

```markdown
<p align="center"><img src="..." /></p>
```

**변경 후:**

```markdown
<div align="center"><img src="..." /></div>
```

### 4. 한 줄 이미지 블록을 여러 줄로 포맷팅

코드 블록(```) 외부에서 `<div>` 한 줄 안에 `<img>`가 함께 작성된 경우 여러 줄로 분리한다. `<img>` 태그의 들여쓰기는 **스페이스 2칸**으로 한다.

**변경 전:**

```markdown
<div align="center"><img src="..." width="80%" class="image__border" /></div>
```

**변경 후:**

```markdown
<div align="center">
  <img src="..." width="80%" class="image__border" />
</div>
```

### 5. `<img>` 태그 들여쓰기 정규화

이미 여러 줄로 작성된 이미지 블록에서도 `<img>` 태그의 들여쓰기가 **스페이스 2칸**이 아닌 경우 2칸으로 통일한다.

**변경 전:**

```markdown
<div align="center">
    <img src="..." width="80%" class="image__border" />
</div>
```

**변경 후:**

```markdown
<div align="center">
  <img src="..." width="80%" class="image__border" />
</div>
```

### 6. `<br/>` 삽입

`</div>` 다음의 **첫 번째 비어있지 않은 줄**을 확인한다.

#### 6.1. `<center>` 출처 설명이 있는 경우

`</div>` 다음에 `<center>` 태그로 감싼 출처 설명이 있으면, `</center>` 이후를 처리 대상으로 삼는다.

**변경 전:**

```markdown
</div>
<center>마이크로소프트-.NET 마이크로 서비스 - 아키텍처 eBook</center>

본문 내용이 이어진다.
```

**변경 후:**

```markdown
</div>
<center>마이크로소프트-.NET 마이크로 서비스 - 아키텍처 eBook</center>

<br/>

본문 내용이 이어진다.
```

#### 6.2. 일반 텍스트가 이어지는 경우

`</div>` 또는 `</center>` 다음의 **첫 번째 비어있지 않은 줄**을 확인한다.

| 조건 | 처리 |
|------|------|
| `#`으로 시작하는 헤더 | 변경 없음 |
| 줄 전체가 `<br/>` | 이미 존재함, 변경 없음 |
| `<div`, `<p` 등 다른 HTML 블록 태그로 시작하는 줄 | 변경 없음 |
| 그 외 (일반 텍스트, `-` 리스트, 숫자 리스트 등) | `<br/>` 삽입 |

`</div>` (또는 `</center>`)와 본문 내용 사이에, 빈 줄을 사이에 두고 `<br/>` 단독 줄을 삽입한다.

**변경 전:**

```markdown
<div align="center">
  <img src="..." />
</div>

본문 내용이 이어진다.
```

**변경 후:**

```markdown
<div align="center">
  <img src="..." />
</div>

<br/>

본문 내용이 이어진다.
```

`</div>` 다음에 빈 줄 없이 바로 본문이 이어지는 경우도 동일하게 적용한다.

**변경 전:**

```markdown
</div>
본문 내용이 이어진다.
```

**변경 후:**

```markdown
</div>

<br/>

본문 내용이 이어진다.
```

### 7. 파일 저장

변경된 내용을 `PATH` 파일에 저장한다. 변경이 없었던 경우에도 동일하게 보고한다.

## 주의사항

- **코드 블록(```) 내부는 절대 변경하지 않는다.**
- **인용문(`>`) 내부는 절대 변경하지 않는다.**
- **YAML front matter는 변경하지 않는다.**
- `<img>` 태그를 포함하지 않는 일반 `<div>` 또는 `<p>` 블록은 처리하지 않는다.
- `</div>` 다음에 `<center>` 태그(출처 설명)가 있으면, `<center>` 블록을 건너뛰고 `</center>` 이후를 기준으로 `<br/>` 삽입 여부를 판단한다.
- 모든 실행은 한국어로 수행한다.

```
